# 인터넷공룡 운영 전환 체크리스트

| 구분 | 확인 항목 | 완료 기준 |
| --- | --- | --- |
| 인증 | `/admin/login` 접속 가능 | 로그인 화면 정상 노출 |
| 인증 | Supabase Auth 또는 Preview Login 설정 | 운영 진단에서 인증 모드/설정 정상 표시 |
| DB | `SUPABASE_SERVICE_ROLE_KEY` 입력 | 운영 진단에서 DB 연결 정상 |
| 데이터 | 상품/게시물/후기/문의 건수 확인 | 운영 진단 데이터 건수 확인 |
| 콘텐츠 | 가상 데이터 교체 여부 | 실제 운영 데이터 반영 완료 |
| 문의 | 메인/상품/신청 폼 저장 확인 | 관리자 문의 목록에서 확인 |
| 정책 | 개인정보처리방침/이용약관 점검 | 최소 뼈대 반영 |

## 운영 전환 순서

| 순서 | 작업 | 비고 |
| --- | --- | --- |
| 1 | `.env.local`에 실제 도메인, Supabase env 입력 | Node 20 기준 권장 |
| 2 | `db/supabase-schema.sql` 적용 | 테이블 및 정책 생성 |
| 3 | 관리자 계정 생성 및 `ADMIN_ALLOWED_EMAILS` 등록 | Supabase Auth 기준 |
| 4 | `npm run doctor` 실행 | env/DB/데이터 소스 상태 사전 점검 |
| 5 | `npm run seed` 또는 실제 데이터 import 진행 | 운영 데이터 반영 |
| 6 | `/admin/system`에서 상태 확인 | 경고/미설정 항목 제거 |
| 7 | 실제 문의 테스트 진행 | 문의 저장 및 관리자 반영 확인 |

## 외주 1차 기준 메모

| 항목 | 현재 구현 | 후속 고도화 |
| --- | --- | --- |
| 인증 | Supabase Auth + Preview fallback | MFA, 역할 분리 |
| 운영 진단 | 관리자 화면 내 수동 점검 | 자동 알림, 외부 모니터링 |
| 데이터 전환 | seed/import 스크립트 | CMS 또는 운영툴 일원화 |
