# 인터넷공룡 홈페이지 1차 디자인 방향서 및 텍스트 와이어프레임

## 0. 문서 개요

| 항목 | 내용 |
| --- | --- |
| 문서 목적 | 실제 시안 작업 전, 디자인 톤과 화면 구조를 클라이언트와 선합의하기 위한 1차 방향 문서 |
| 기준 문서 | `인터넷공룡 홈페이지 선수금 수령용 1차 기획안` |
| 서비스 성격 | 인터넷/TV/통신 가입 상담 및 상품 비교/신청 유도형 홈페이지 |
| 디자인 핵심 | 신뢰감 있는 비교 상담 서비스 톤, 전환 중심 레이아웃, 모바일 우선 대응 |
| 레퍼런스 반영 원칙 | CTA 반복 구조와 비교/상담 흐름은 참고하되, 화면 구성과 톤은 인터넷공룡에 맞게 재설계 |
| 비주얼 원칙 | 흰색 기반의 밝은 인상 + 주황색으로 전환 강조 + 연보라/하늘색으로 정보 레이어 분리 |

## 1. 브랜드 해석

| 항목 | 해석 |
| --- | --- |
| 브랜드명 의미 | `인터넷공룡`은 “인터넷/통신 영역을 잘 아는 크고 든든한 전문가”라는 인상을 만들 수 있는 이름이다. |
| 사용자 인식 목표 | 어렵고 복잡한 통신 가입 과정을 대신 정리해주고, 조건 비교를 도와주는 상담 브랜드로 인식되도록 설계한다. |
| 톤 앤 매너 | 친근하되 가볍지 않고, 설명은 쉽게 하되 상담 전문성은 분명하게 드러나는 방향이 적합하다. |
| 캐릭터 사용 원칙 | 공룡을 전면 캐릭터로 내세우기보다, 둥근 실루엣, 발자국 모티프, 작은 아이콘 포인트 정도로 절제해 사용한다. |
| 피해야 할 방향 | 아동용/키즈 서비스처럼 보이는 과한 캐릭터성, 과장된 말풍선, 장난감 같은 색 사용, 지나치게 만화적인 일러스트 |
| 브랜드 인상 문장 | “비교는 명확하게, 상담은 친절하게, 가입은 안심되게.” |

## 2. 디자인 컨셉 키워드 3개

| 키워드 | 의미 | 화면 적용 방식 |
| --- | --- | --- |
| 믿음 가는 친절함 | 딱딱한 통신 판매 사이트가 아니라 설명을 잘해주는 브랜드 인상 | 넉넉한 여백, 짧고 명확한 카피, 부담 없는 폼 구조 |
| 비교가 쉬운 구조 | 상품이 많아도 사용자가 헤매지 않도록 정보 위계를 분명히 함 | 카드형 비교, 핵심 수치 강조, 구획이 분명한 섹션 레이아웃 |
| 밝고 정돈된 에너지 | 활기 있는 브랜드지만 싸구려 느낌 없이 가볍고 선명한 톤 유지 | 흰 배경 중심, 주황 CTA, 연보라/하늘색 보조 배경, 가벼운 그림자 |

## 4. 타이포그래피 방향

| 구분 | 방향 |
| --- | --- |
| 본문 기본 폰트 | `SUIT Variable` 또는 동급의 정돈된 한글 산세리프 사용 |
| 숫자/혜택 강조 폰트 | 영문 숫자 강조가 필요한 영역은 `Outfit` 또는 동급 폰트 병행 가능 |
| 헤드라인 톤 | 짧고 단정하게 끊는 문장형 헤드라인 사용. 과장형 광고 문구보다 “비교해드립니다”, “상담 도와드립니다” 같은 신뢰형 표현 우선 |
| 제목 크기 | 데스크톱 H1 44~52px, H2 30~36px, H3 22~26px 권장 |
| 본문 크기 | 데스크톱 16~18px, 모바일 15~16px 권장 |
| 줄간격 | 본문 1.6 전후, 헤드라인 1.25~1.35 전후 |
| 글자 굵기 | 제목 700~800, 본문 400~500, CTA/중요 정보 600~700 |
| 문장 스타일 | 짧고 해석이 빠른 문장 중심. 요금, 혜택, 설치, 결합 같은 단어는 시각적으로 쉽게 찾히게 구성 |

## 5. 레이아웃 및 반응형 방향

| 항목 | 방향 |
| --- | --- |
| 레이아웃 구조 | 카드형 섹션을 기반으로 한 단정한 세로 흐름. 복잡한 3단 구조보다 1단 또는 2단 중심 |
| 데스크톱 기준 폭 | 메인 컨테이너 1200~1280px 권장 |
| 모바일 기준 | 360px 이상에서 무리 없이 읽히는 1열 중심 구조 |
| 전환 요소 배치 | 첫 화면, 상품 카드 하단, 상세 중간, 콘텐츠 하단, 모바일 하단 고정 바에서 CTA 반복 |
| 스크롤 리듬 | `설명 -> 비교 -> 신뢰 -> 문의` 순서가 반복되도록 섹션 간 목적이 분명해야 함 |
| 모바일 대응 | 메인 폼은 상단에서 바로 보이게, 긴 비교표는 카드 스택 또는 아코디언으로 변환 |
| 고정 요소 | 모바일 하단에 `빠른 상담` / `전화 상담` 2버튼 고정 바 권장 |

## 6. UI 컴포넌트 시스템

### 6-1. 공통 컴포넌트

| 컴포넌트 | 구성 | 스타일 방향 | 사용 위치 |

| 상단 헤더 | 로고, 메뉴, 대표 전화 CTA | 흰 배경 + 얇은 보더 + 우측 강조 버튼 | 전체 |
| 히어로 타이틀 블록 | 소제목, 메인 헤드라인, 서브카피, 핵심 배지 | 넓은 여백, 두 줄 내외 카피, 좌측 정렬 | 홈, 통신사 상세 |

| 상품 카드 | 상품명, 핵심 조건, 혜택, CTA | 흰 카드 + 상단 배지 + 하단 고정 CTA | 비교, 추천 영역 |
| 비교 테이블 | 항목명, 통신사별 값, 강조 셀 | 모바일에서는 카드 분리형 | 상품 비교 |
| 뱃지/칩 | 추천, 결합, 인기, 실속형 등 | 연보라/하늘색/주황 톤 분기 | 카드, 필터 |
| 후기 카드 | 제목, 요약, 유형, 평점성 문구 | 실사 없이도 신뢰가 나는 텍스트 중심 카드 | 후기 |
| 게시글 카드 | 카테고리, 제목, 요약, 날짜 | 정돈된 리스트/카드 혼합형 | 게시판 |
| FAQ 아코디언 | 질문, 답변, 토글 | 하늘색 또는 회색 배경 구분 | 상세, 게시판 |
| 하단 고정 CTA | 빠른상담, 전화상담 | 모바일에서 전체 폭 2분할 | 모바일 공통 |

## 7. 메인 화면 섹션 구성

| 순서 | 섹션명 | 목적 | 핵심 요소 | 디자인 메모 |
| --- | --- | --- | --- | --- |
| 1 | 헤더 | 브랜드/탐색/전화 진입 | 로고, 메뉴, 대표번호 버튼 | 우측 전화 CTA는 주황색 강조 |
| 2 | 히어로 + 빠른 문의 | 첫 화면 전환 확보 | 헤드라인, 서브카피, 신뢰 배지, 이름/연락처 폼 | 좌측 메시지, 우측 폼 또는 모바일 세로 스택 |
| 3 | 대표 비교 진입 | 탐색 시작점 제공 | 통신사별 대표 카드 3~4개 | 카드마다 요약 혜택과 CTA |
| 4 | 상황별 추천 | 사용 시나리오 기반 진입 | 이사 예정, 약정 만료, 1인 가구, 가족 결합 | 연보라/하늘색 배경 박스 활용 |
| 5 | 왜 인터넷공룡인가 | 브랜드 신뢰 확보 | 비교 상담, 빠른 회신, 맞춤 추천, 사후 안내 | 숫자보다 메시지 중심 |
| 6 | 대표 상품 요약 | 실제 문의 가능한 대표 상품 노출 | 월 요금, 추천 대상, 구성, CTA | 가상 데이터 기반 |
| 7 | 후기 미리보기 | 상담 신뢰 강화 | 후기 카드 3건, 후기 유형 칩 | 텍스트 위주로 정돈 |
| 8 | 가입 가이드/혜택 | 정보성 탐색 유도 | 가이드 카드, 이벤트 카드 | 하늘색 배경 블록으로 분리 |
| 9 | FAQ | 불안 해소 | 자주 묻는 질문 5~6개 | 아코디언 사용 |
| 10 | 하단 문의 섹션 | 스크롤 말미 전환 회수 | 이름/연락처 폼, 전화 CTA, 안내문구 | 메인과 동일한 폼 재노출 |
| 11 | 푸터 | 정책/사업자 정보 | 약관, 개인정보처리방침, 사업자 정보 | 신뢰 마감 영역 |

## 11. 시안 작업 시 유지해야 할 체크포인트

| 체크 항목 | 기준 |
| --- | --- |
| CTA 가시성 | 첫 화면 안에 반드시 1개의 주요 CTA와 1개의 보조 CTA가 함께 보여야 함 |
| 색상 사용 | 주황색은 반드시 전환 행동에 우선 사용하고, 장식용 남용 금지 |
| 콘텐츠 밀도 | 한 섹션에 메시지 1개만 명확히 전달. 텍스트 과다 금지 |
| 공룡 모티프 | 아이콘/포인트 수준으로 제한, 대형 캐릭터 노출 지양 |
| 모바일 가독성 | 비교표, 카드, 폼 모두 1열 기준으로 다시 읽히게 설계 |
| 폼 피로도 | 메인 폼은 2개 입력 유지, 상세 폼도 단계가 많아 보이지 않게 그룹화 |
